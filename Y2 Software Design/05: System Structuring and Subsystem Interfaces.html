Identify subsystems and interfaces
  • Identify major functions system required to provide (from use cases).
  • Associate major system functions with subsystems by grouping together logically similar functions.
  • Subsystems may be decomposed further using same principle.

Specifying subsystem Interfaces
Interface name
Unique identifies e.g. ISensor (‘I’ denotes Interface)
Operations
Name of each operation, together with input and return parameters and exceptions e.g. motionDetect(int): none
Exceptions
Name and data context for operations exceptions i.e. what causes it.
Service quality
Optional: Non-functional properties provided at interface.

Ball and Socket Interface notation (alarm example)
  • Provided: —(
  • Required: —O
