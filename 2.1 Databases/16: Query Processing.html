The Process
  1. Scanner: Identifies language keywords
  2. Parser: Checks query syntax to determine if it abides rules
  3. Validation: Checks that all relation / attribute names are valid
  4. Internal representation of query: Usually a tree or graph
  5. Query Optimiser: Devises an execution plan for the result
  6. Code Generator: Takes the plan and makes the code
  7. Runtime database Processor: Runs the boi
Optimisation Techniques
Cost
Components 
  1. Storage: Storing intermediate files generated by execution strategy for query
  2. Computation: Performing operation on data buffers
  3. Memory: Retaining number of memory buffers needed for query
  4. Communication: Shipping query results from DB to client

Aims determined by scenario
  • Large: Minimise access cost to secondary storage to minimise block transfers between disk and main memory.
  • Small: Minimise computation cost
  • Distributed: Minimise communication cost
Semantic
Able to set constraints on the database, thus if querying something that violates there’s no need to execute as it will return nothing
  • E.g. an employee canny earn more than their manager
  • Thus if the max salary by manager is £50,000 then no need to search for an employee that earns £55,000
Heuristic
Done via query trees, some general rules include
  • Select and project operations are moved down the tree
  • Most restrictive joins and selects are also moved down.
  • Must ensure that transformation always lead to equivalent query tree

Language Optimisation
  • Low Level Navigational Languages: Hand optimised by the programmer 
  • High Level Relational Languages: Declarative, care more about the what rather than the how thus requires more optimisation
