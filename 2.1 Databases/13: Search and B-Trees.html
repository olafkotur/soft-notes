Search Tree: Special type of tree used to guide search for a record given value of one of record’s fields.
  • Uses a default scheme of search trees where all the lower children are to the left of the parent and the higher on the right.
  • Important to keep the tree balanced - have all of its leaf nodes at same level to reduce number of block accesses.
  • When records deleted it may leave some nodes nearly empty thus wasting storage.
  • Often gets unbalanced.






B-Tree (balanced): Displays all the keys that are less than or higher than the parent key
  • Takes into account all combinations of the keys
  • Guarantees that tree is always balanced when we insert/delete.
  • Each field contains the key and the pointer to the block or cluster of blocks



Adding to B-Tree
  • We’re able to choose which branch we add the element to
  • Thus always keeping the tree balanced
  • When the page in the leaf becomes too large we are able to split this and continue to have a balanced tree.

Insertion
  1. Node is split into two nodes at same level
  2. Middle entry moved to parent node along with two pointers to new split nodes
  3. If parent is full it is also split
  4. Splitting therefore can go all the way back to the root which means that the root can also be split - this is what guarantees that the tree is always balanced.




B+ Tree: Variation of B where you only store pointers to blocks in the leaf nodes.
  • Each node has information on the lowest value for each leaf node
  • We’re able to add, move and delete leaf nodes at all times

Insertion
  1. Try to add to the left node
  2. Try to add to the right node 
  3. Split the node
