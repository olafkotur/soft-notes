Relational Algebra: Basic set of relational operations to retrieve, filter and restructure relations where the result of operations is always a new relation.




SELECT Operation ( sigma: σ )
  • Order of conditions doesn’t matter.
  • Will return any rows where the condition/s are true.
  • Multiple SELECTs can be combined to a single query
SELECT * FROM STUDENT WHERE FName = “Bart” AND GENDER = “M"


PROJECT Operation ( pi: π )
  • Will return a non duplicate project of the first name and last name.
  • The order of the conditions matter.
  • May not always return the same number of rows as you started with as it pulls unique names only.
  • Only returns the columns specified in the attribute list.
SELECT FName, LName FROM STUDENT


RENAME Operation ( rho: ρ )
  • Rename or name the result of a relation
SELECT FName AS “First Name”, LName AS “Last NAME” FROM STUDENT


Union, Difference, Intersection Operations



Concatenation
CONCAT(employee1, project1) = (Smith, 1, 1, Beta)



JOIN Operation ( ⟗ )
  • Left / Right outer join retains everything in the left or right side of the join.
  • Denoted as R ⟗ S  ||  S ⟗  R
  • EQUIJOIN to apply a requirement to the join e.g. ON A.DOB = B.DOB
  • NATURAL join returns all the attributes that are not in JOIN

OUTER JOIN
  • Keep every tuple in relation R
  • LEFT, RIGHT, FULL depending on which relation you would like to keep tuples from.
  • Keeps all tuples in dominant relation even if there is no match in the second relation.
