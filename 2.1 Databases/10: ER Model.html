Entity Relational Model: Allows analysis, specification and design of database structure - used in conjunction with a graphical notation. Includes three concepts:
  1. Entities - Data about a specific object, real or abstract.
  2. Relationships - Associations among entities.
  3. Attributes - Properties describing an entity

Relationships

Degree of a Relationship: Number of participating entity types.
  • Two: Binary relationship
  • Three: Ternary relationship
  • n: N-ary relationship


Recursive Relationships: Participating entity types are the same.


Cardinality Ratio: Number of relationships instances as entity can participate in.
  • Cardinality ratio Department : Student is 1:N
    • Means that each department can enrol multiple students while a student can be enrolled by only one department.
  • Can have other relationships such as M : N which means numerous on both sides, or 1 : 1 where only 1 relationship allowed per entity.


Key Attributes: Have unique values for each instance of entity type, also have a different graphical notation:


Atomic Attributes: Indivisible, such as a name or anything that is not a number value e.g. Department Name and Course Name.

Composite Attributes: Can be divided into smaller subparts that represent more basic attributes with indecent meaning
  • Example, Student name can be divided into First Name and Last Name


Multi-valued Attributes: Allowed to have a set of values for a particular instance of entity type e.g. Location can have { Street, City, Country }

ER-to-Relational Mapping
Step 1 Basic
Convert each entity type including all simple attributes of said entity. Then choose on the key attributes as the primary key of relation.
Step 2 1:1 
For each 1:1 binary relationship, choose one of the relations and include primary keys of one as the foreign keys of another.
Step 3 1: N
For each 1:N binary relationships, include primary key of relation T (T:S) as foreign key in S.
Step 4 M:N
For each M:N binary relationship, create new relation to S (T:S). So take the relationship, make a new table and add the attributes to it.
Step 5 Multi
For each multi-valued attribute A create a new relation R including an attribute corresponding to A, include simple if A is composite. The primary key will be the combination of A and K.
