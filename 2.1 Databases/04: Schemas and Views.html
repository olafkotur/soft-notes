Schemas: Structure of a database which allows user to:
  1. Declare types of records required e.g. Person, Company...
  2. Declare types of relationships among records, e.g. Person works for company, company employee person.

Three Levels (Schemas) of Data 
  1. Internal Level
  2. Conceptual Level
  3. External View
Internal Level: Uses physical data model model, describes details of data storage and access paths for database.
  • Internal schema that defines physical implementation of database (storage files contain actual data records).
  • Defines database as collection of logical records i.e. internal names of fields and number of bytes they occupy.
Conceptual Level: Defines entire information resource at an abstract level, for a community of users.
  • Hides details of physical storage structures, concentration on:
    1. Entities represented in databas.
    2. Relationships between entities
    3. Data types
    4. Security restrictions
    5. Audit controls
    6. Contraints
    7. Validation procedures
  • High level data model e.g. Relational model is used here.
External View: Hides rest of database from user as they are not interested in it or not allowed to access it.
  • Each external schema describes database view of one group of database users. 
  • Each view is subset of conceptual level, defined and formatted according to user requirements. 


Three-Schema Architecture
  • Even though there are multiple layers it will always come down to the internal level to retrieve data.
  • For this reason if a request is made at external then it must be transformed to conceptual level which will then transform to physical level.
  • Use Mappings to achieve this - however, often time-consuming and resource intensive.

Views: Separately generated glimpse of a database custom to the application.


Physical Data Independence: Capacity to change internal schema without having to change conceptual or external schemas.

Logical Data Independence: Capacity to change conceptual schema without having to change external schema or application programs e.g. adding a new attribute to existing schema.
  • More difficult to achieve when the change is subtractive.
