Second Normal Form
  1. Must be in first normal form
  2. AND every nonprime attribute A in R is not partially dependent on any key of R.

Prime Attribute: Attribute that is a member of some candidate key of R

Dependancy
Deletion
Deleting only record of information when wanting to delete one attribute in a domain.
Update
When we want to update one record in a domain we would have to update ALL records to maintain consistency.
Functional
Each value of attribute A has no more than one value for attribute B in a relation e.g. If we queried for Supplier = “Ajax” then it would return the Stock = { 25, 18 } thus doesn’t have functional dependency i.e. A —> B

Full-functional Dependence
Catalogue (Supplier, Part#) —> Catalogue (Stock)
  • To get the result for a single row we need to query as above ^
  • However, if we drop one of the attributes either Supplier or Part# then the query may result in multiple rows being given.

Therefore….
  1. Stock is dependant on all of { Supplier, Part# }
  2. Stock is not dependent on Supplier
  3. Stock is not dependent on Part#
  4. Hence, Stock is fully dependent on combination of Supplier and Part#



Solution: Replace the relation with two new relation by projecting out dependencies worked out.
S = PROJECT (Supplier, Part#, Stock)
T = PROJECT (Supplier, Location)
