ODBC: Open Database connectivity, common interface for accessing heterogeneous SQL databases.

Interfaces define:
  1. Library of function calls hallowing an app to connect to DBMS, execute SQL and retrieve results.
  2. Standard way to connect and log on to DBMS
  3. Standard representation of data types
ODBC Architecture
Application
Performs processing and calls ODBC functions to submit SQL and retrieve results from DBMS

Driver Manager
Loads and unloads drivers on behalf of app, can process ODBC function calls or pass them to driver.
Driver and Database Agent
Process ODBC function calls, submit SQL requests to specific data source, and return results to application.
Data Source
Data user wants to access, its associated DBMS, its host operating system and network platform, if any.

JDBC: Java database connectivity, developed in 1996. Essentially an API for application writers e.g. SQLite, MySQL, Postgres.
  1. Establishes connection
  2. Sends SQL statements
  3. Processes the results
