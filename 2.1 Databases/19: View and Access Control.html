Assignment
  • You have to select all the info and then create a new table with the desired info
  • Hides all the other info from the DB so user only sees what they need to see
SELECT pname, sname
FROM personel p, ship s, assignment a
WHERE (p.ppid = a.apid)
AND (a.asid = s.ssid);



Creating a View
  • Able to do everything to a view as you usually would to a normal relation
  • Also hides complexity from users, thus acts as a form of abstraction
CREATE VIEW personel_on_ship AS

SELECT pname, sname
FROM personel p, ship s, assignment a
WHERE (p.ppid = a.apid)
AND (a.asid = s.ssid);

Access Control
Mandatory
Each database object is assigned to a classification level, includes:
  1. Top Secret
  2. Secret
  3. Confidential
  4. Unclassified
Discretionary
Each user is given appropriate access rights on specific database objects

Authorisation Identifier: Used to establish identity of a user in SQL e.g. username and password
  • Given that every query is ran by a user, we can determine which objects can be accessed

Privileges: ISO Standard defined privileges:
  1. Select: Retrieve data from table
  2. Insert: Insert new rows into table, can be restricted to specific columns
  3. Update: Modify rows of data, can be restricted to specific columns
  4. Delete: Delete rows of data from table
  5. References: References columns of table in integrity constraints, can be restricted to specific columns

Granting Privileges
GRANT {PrivilegeList | ALL PRIVILEGES} ON ObjectName
TO {AuthorizationList | PUBLIC}
[WITH GRANT OPTION]

// Example
GRANT ALL PRIVILEGES
ON Staff
TO Manager
WITH GRANT OPTION


Revoking Privileges
REVOKE {PrivilegeList | ALL PRIVILEGES} ON ObjectName
FROM {AuthorizationList | PUBLIC}

// Example

REVOKE SELECT
ON Branch
FROM PUBLIC

REVOKE ALL PRIVILEGES
ON Staff
FROM Director
