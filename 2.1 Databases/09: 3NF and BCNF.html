Transitive Dependency: When an attribute depends a non-primary key within a table
  • The solution to this is to create a new table for the attribute that is dependent
  • Add foreign keys to these tables to be able to link it together

Transitive Dependence
R(A) --> R(B)
R(B) -/> R(A)
R(B) --> R(C)

// Thus
R(A) --> R(C)
R(C) -/> R(A)

Strict Transitive Dependence
R(A) --> R(B)
R(B) -/> R(A)
R(B) --> R(C)
R(C) -/> R(A)



Boyce-Codd Normal Form: Stronger form of 3NF where every determinant must be a super key.
  • In other words, a non-prime attribute should not be used to access a prime attribute.
  • Instead the prime attribute should lead to a non-prime attribute
