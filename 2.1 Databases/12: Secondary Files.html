Secondary Files: Additional files on top of the primary files, used to speed up records in response to certain search conditions.
  • Alternative ways of accessing records without affecting the physical placement of records on disk.
  • Indexes are in order so that you are able to do a binary chop on the index.


Ordered Files - Indexes
Primary
File is ordered according to a field e.g. employee name. Index is also sorted by the same field. The first record in each block is the block anchor.

Secondary
Able to have multiple secondary indexes - in addition to the primary index we are able to create for each specific search field.

Multiple appearances of secondary key value
  • Solution 1: Point to specific records rather than data blocks
  • Solution 2: Indirection, point to a block of record pointers i.e. have a thing in between the index file and data file. 
Clustering
Used if ordering field is a non-key field e.g. numerous records have same value for ordering field. There is only one index file for each distinct value of clustering field.

Contains two fields…
  1. First field is the same data type as clustering of the file
  2. Second field is the pointer to the disk block, the pointer will only point to a disk block that has that value as a first value in its field.
 
Multi-level
Create indexes on indexes which helps to reduce search space - able to do this on any type of index.
