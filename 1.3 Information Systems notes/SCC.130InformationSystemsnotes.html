IS: Transmission medium

Bandwidth

  • Range of frequencies that a channel or medium can allow in Hz.
  • Bandwidth in bits per second refers to rate of bit transmission that a channel can transmit.
  • Bandwidth of telephone is above 3,000 Hz for voice. If digitally transmitted it can have rate of 48 kbps. If reduced to 2,400 Hz then transmission is 38.4 kbps.
  • Reason for not expanding bandwidth is because it would be too expensive & would cause traffic in the network.
  • Data is sampled at one end, and reconstructed at the other end through the use of a filter. The more data points, the more accurate we can be in data transmission.
  • 1 sample is 8 bits, therefore, bit rate is simply how many samples per second.

Throughput

  • How fast we can send data through network, always less than bandwidth.

e.g. may have bandwidth of 1 Mbps, but device can only handle 200 kbps.

e.g. motorway designed to transmit 1,000 cars per minute, if congestion, then reduced to 100 cars per minute.  

If bandwidth  passing average of 12,000 frames (10,000 bits each) per minute .˙.

Latency

  • Delay, how long it takes for entire message to completely arrive at destination from time of the first bit sent.

  • Propagation time measures time required for electrical signal travel from source to destination () e.g. .˙.
  • Transmision time is time for transmitter to emit single frame, depending on size of message and bandwidth of channel at transmitter ()

e.g. message size is 2.5 kbyte, and bandwidth of TX network is 1 Gbps .˙.

IS: Data communication

Effective Data Communication

Delivery: correct destination. Data cannot be accessed by an un-intended user.

Accuracy: Altered data during transmission is unusable. Result of distortion or noise.

Timeliness: Late data is of no use (packets usually re-sent if no acknowledgement received. Some delay will occur, but time relationship remains constant e.g. 1m/s.

Jitter:  Variation of packet arrival time e.g. 1st packet arrives with 1s delay, 2nd with 5s, 3rd with 7s delay.

Data Flow

  1. Simplex - data travels only in one direction e.g. mainframe → monitor.
  2. Half-duplex - data travels both directions but at separate times

Local Area Network (LAN)

Privately owned, linking devices to a single office. Can be as simple as 2 PCs. Limited to a few km. Most LANs use connecting switch to route traffic to desired destination. Deals with devices close together.

Metropolitan Area Network (MAN)

Can extend over entire City (TV Cable). Either owned and operated privately and publicly. Multiple devices can be connected e.g. a telephone box will have multiple houses connected to it.

Wide Area Network (WAN)

Long-distance transmission (unlimited) of data, voice, image and video. WAN interconnects connecting devices (switches, routers modems). Usually run by communication companies, WAN owned by 1 company known as Enterprise Network.

Internetworks

  • Rare to see LAN and WAN isolated, today we often use a combination of both

Time Division Multiplexing (TDM)

Frequency Division Multiplexing (FDM)

Allows a user to use a path for a certain amount of time. After their allocated time is over, another user can switch to using that particular path. Limited as it can take a while before the same user has access to the path again e.g. if a city of 1 million spends 3 minutes on pathway, then a user has to wait 6 years for the next turn.

Allows multiple users to share the same bandwidth by allocating specific carriers within a bandwidth. This means that multiple users can talk simultaneously.

Combination of TDM and FDM is used to achieve the lowest possible traffic across bandwidth and pathways. Multiple FDM channels are used in different areas, then using TDM, each have access to the path for a few milliseconds which allows to control traffic.

General Packet Radio Service (GPRS)

  • If whole bandwidth is unused we can allocate more space for a user to speed up the process.

Circuit-switched networks

  • Consists of set of switches connected by physical links.
  • Connection between two stations is a dedicated path.
  • Each link is divided into a number of channels using either TDM or FDM.

Switched Network Strategies

Circuit Switching (data flows continuously, no messages or packets e.g. oil pipeline)

  1. Dedicated circuit established across sequence of links.
  2. Path exists at both ends until call is finished.
  3. Several circuits share link, but each allocated fixed amount of link capacity.
  4. If circuit unused (silent telephone conversation) link capacity is wasted.
  5. If link capacity reached, new circuits cannot be established.

Packet Switching (data is routed by router, e.g. bus)

  1. Nodes (routers) divide data into discrete ‘blocks’ or ‘packets’.
  2. Each packet sent separately into network.
  3. Packets have header (destination info).
  4. Packets follow different paths from source to destination - if one path has too much traffic, router will send packet a different route.
  5. Packet may get lost (resent if TCP, ignored if UDP)

User Datagram Protocol (UDP)

Send packet and forget about it (unreliable)

Transmission Control Protocol (TCP)

Ordered delivery of data, sends acknowledgement when received (reliable) However:

  • If we send 10 packets instead of 2, less risk of corruption, only lose 1/10 instead of ½. Congestion is also less impactful.

e.g. when you are sending a letter to a friend, you go through a series of protocols rather than being present at each step of the process.

The relational data model

  • Edgar Codd, 1923-2003
  • Based on maths theory and set of relations and predicate logic.
  • Data visualised in tables, but arranged in sets of tuples with common attributes.

Integrity

Domain Integrity

  • Each attribute (column) has a name and domain.
  • Domain specifies data type and range and default value
    e.g. INTEGER (INT), DATE, VARCHAR(n)

  • NULL represents ‘no data’ - attributes defined as NOT NULL cannot be NULL (enforce data entry).

Entity Integrity

  • Keys underpin relational model - represent attribute (or combo) used to uniquely identify tuples in relation (no such thing as row number, only unique ID).
  • Primary keys uniquely identify tuples in relation (can’t be NULL).
  • Foreign keys cross-references data between relations.
  • Candidate key is any unique identifier of a tuple, cannot be referenced elsewhere.

  • Integrity between referencing other tables e.g. can’t make changes in one table without referencing tables also updating → leads to referential integrity violation.

DB: The Database Management System (DBMS)

File based systems

  • Data stored in different files, each with specific format, each different application must understand the format of the files that it’s using.
  • Lots of room for human error and other issues with data including:
  1. Repeated data
  2. Referenced data
  3. Incorrect data
  4. Multiple locations of data

Database provides...

  • Reliable data storage accessible by users maintaining privacy and data integrity.
  • DBMS is a software that implements a database providing:
  1. Data Definition Language (DDL) → Structure in which data is conceptually stored. A table is called is a scheme, a group of tables, schemas.
  2. Data Manipulation Language (DML) → Operators e.g. insert, delete, select
  3. Data Control Language (DCL) → Control access to the database

Structured Query Language (SQL)

  • Declarative non-procedural language where we say what we want and not how it’s supposed to be done.

ANSI-SPARC Architecture

  • DBMS framework developed in 1975.
  • External level → different views of database tailored to users’ needs.
  • Conceptual level → organisation of entire database contents, i.e. what data is stored and how including relationship between data and its constraints.
  • Internal level → physical data storage, how data is described, indexed and ordered.

Data independence

  • Logical: Capacity to change the conceptual schema without having to change the external schemas or application programs e.g. adding new column to table doesn’t affect view of existing columns.
  • Physical: Capacity to change the internal schema without having to change the conceptual schema e.g. changing of storage devices does not affect structure of relations.

SOC: Copyright laws and Software protection

Intellectual property

  • Any unique product of human intellect that has commercial value e.g. books, songs, paintings, chemical formulas, computer software.
  • Owner has natural right to the own the property, supported by John Locke’s theory.
  • This gives inventors exclusive rights to their idea but only for a finite period of time, after which they must apply for a patent.
  • Companies can use trademarks, signs which are capable to graphically represent goods or services e.g. Apple Logo.

Copyright law

  • Provides the owner with exclusive rights to use the work.
  • Other people or companies can use the work of the original owner if they have their consent to do so.
  • Any unauthorised use of work is classed as a copyright infringement.

Patent law

  • Intellectual property on drugs, gives the owner the exclusive right of an invention for up to 20 years.
  • Protects against independent development of something based on the same idea.
  • Takes a long period of time and money in order to gain a patent (UK roughly 3 years).

Law of Confidence

  • Protects information with a very wide scope e.g. trade secrets, lists of clients and contacts, ideas not yet expressed in a tangible form.
  • Flexible and adaptable to technological advances but limited to information of confidential nature.

Online Piracy

  • Pirate: Someone who a person believes is unfairly trading on their creation.
  • Piracy: Act of unfairly trading on a person’s creations
  • Example: Kim Dot Com, owner of file-hosting site MegaUpload where users could share files for free, later used for piracy as it was an easy way to send files to others.

Combating Piracy

  • Digital Rights Management (DRM): Access control mechanisms used to restrict medium usage by inhibiting usage by multiple parties.
  • Web-tracking: Digital Economy Act 2010, holders send IP addresses of downloaders to ISP who then contact the customers.

Free Software: Open Source

  • Freedom to run the program, adapt the program and redistribute the program.
  • Programmers develop software in return for credit enhancing their reputation and make grants easier to acquire.
  • Shareware suggests that you can download software for free but should pay the developer a small reward e.g. Humble Bundle.
  • Copyrights, Designs and Patent Act 1988 protects those that do want a payment in return for their software, applies same rules as Copyright Laws.

SOC: Malware

Taxonomy of Malware

Trojan

Malicious software masquerading as something else. Normally used to steal information and gain control.

Back Orifice, 1998, user installs server-side program, 3rd party can control and access machine.

Virus

Embedded program that copies itself to other programs. Resident viruses copy into memory and run in background. Non-resident viruses find hosts to infect and spread.

Conficker, 2008, 15,000,000 computers infected due to Win32 weakness to create remote controlled botnet.

Worm

Copies itself across a network, doesn’t need to infect across a network, often spreads across emails.

Stuxnet, 2010, created by Israel and US to attack Iran’s nuclear facilities.

Zombie

Groups of machines leased out to 3rd parties for flooding servers with requests leading to denial of service (DDoS) or to send out widespread spam

Botnet, 2010, sent Trojans in email attachments, had 30m machines and sent out 3.6b emails per day.

Ransomware

Encrypts all user data and forces users to pay ransom to get key.

NHS, May 12 2017, affected 150 countries and 200K+ machines. Required $300 first 3 days, then $600 after 6, otherwise deleted.

Gaining access

  1. Installed on a disk where autorun is enabled.
  2. Bad link in an email or a website
  3. Malicious code inserted by hacker
  4. Exploitation of OS weaknesses
  5. User installs it unknowingly

Effects of Malware

  • Breaks operating system features such as not being able to access anti-virus scan software online, can also create new directories and allow pop ups.
  • Generates large volumes of network traffic known as a DDoS.
  • Keystroke logging to steal personal information.
  • Computer is recruited into a Botnet and leased out to 3rd parties.

Malware Practitioners

  • Criminals: Stealing personal data for malicious use, can lead to identity theft.
  • Script kiddies: Hack for the fun of it or curiosity to test vulnerabilities.
  • Anarchists: DDoS against government agencies e.g. Anonymous.
  • Government agencies: Such as GCHQ with their Rolling Thunder DDoS.

Malevolent Online Practices

  • Phishing: Spam email communication containing link to be accessed.
  • Social Engineering: Collect data about a person (rubbish, social media, shoulder surfing). Nigerian Prince asks for money or via dating site then asks for money. Can also pose as somebody else e.g. changing personal data over the phone.
  • Grooming: Gaining trust by being nice to victim, may reveal information such as personal data for online banking or involve them in a crime unknowingly.
  • Online Stalking: Lateral surveillance, act of viewing person without them knowing, involves checking a prospective dates and employees.

Ethical dilemmas

  • Hacking: UK Government spent £8.8b on Olympics through private company and denied information requests, thus public will never know how money was spent.
  • Personal data: One may have an application that searches for people, grouping many websites together. Should this sort of data be released to public?

Liability

  • Product: Consumer protection act 1987 (CPA) imposes liability on the producer of a defective product → applies to hardware but not software.
  • Contractual: Supply of Goods and Services Act (SGSA) binds software supplier to provide service with reasonable care. Doesn’t apply to software sold without some form of contract.
  • Negligence: Imposes liability on person that acted carelessly under common law. Any form of failure to adhere professional standards is negligence e.g. sharing personal data or voicing opinions on social media.

Software errors

  1. Type conversion, overflow, truncation: Mars Climate Orbiter 1999, Lockhead Martin calculated thrust in foot pounds but Jet Propulsion Lab calculated in newtons. Orbiter approached Mars at 4.45 times the speed #toast.
  2. Race conditions and concurrent tasks: Amazon price war between bots, one book was undercutting another book that always set its price higher, price peaked at £23m.
  3. Incorrect synchronisation: Postmasters wrongly accused of false accounting when in reality there was an issue with the Horizon system.

Data errors

  1. Entering wrong data: Dutch chemical plant explosion 1992 due to single mistyped character as worker entered ‘tank 634’ instead of ‘tank 632’.
  2. Incorrect data manipulation: Ariane 5 launch 1996 ($500m), self destructed 40 seconds after launch as an exception occurred without a way to handle it.

Three laws of Robotics

  2. Robot must obey orders given to by human beings except where such orders conflict with the first law.
  3. Robot must protect its own existence as long as it does not conflict with first and second law.

SYS: Disaster Recovery

Disaster recovery planning

  • Designed to reduce likelihood of a disaster but recognises that disasters do occur, when that happens we must look at limiting its impact.
  • Must identify and protection of an organisation’s critical processes to main continuity.

e.g. In 2015, storm Desmond has flooded the generator in Lancaster causing 55,000 homes not having power, this included the University where it had to send its students home.

     

Backup systems

  • Synchronous replication: Remote copy of data created simultaneously as primary site is updated. After disruption, operations can resume almost immediately.
  • Distance: Latency increases with distance, delays in propagating updates can affect performance and reliability e.g. host must get acknowledgement.
  • Asynchronous replication: Covers any distance without affecting application performance since no propagation delay involved.

Mechanisms to ensure business continuity

  • Analysis capability: How your organisation would currently respond to disaster.
  • Analysing risk: How different disaster scenarios would affect organisation.
  • Recovery time: Time from data loss to recovery.
  • Recovery point: Acceptable amount of data loss.
