Patterns of Human Behaviour 
Goal. Execute, Evaluate
Form a goal then choose and execute an action, observe the result and repeat until goal is reached.
Unit Task
Tasks for which everything is directly accessible i.e. working memory or in front eyes.
Tool Focus
Normally, we devote very little attention to the tools used to perform the task. When tools draws attention, it distracts from our goals.
Scent of Information
Selective attention depending on our goals, users will not check all options but make choices based goals.
External Aids
Counting: Move counted, point to last counted, count on fingers.
Reading: Insert bookmark to show last read page.
Checklists: Keep track of short-term tasks.
Familiar Paths
Exploring new path is problem-solving and attention demanding, taking familiar routes can be done automatically.
Cleanup
Once goal is ached, everything related to it is dropped from working memory - people don’t cleanup at end of task e.g. leaving credit card at shop.


Nielsen’s 10 Design Heuristic
Control and Freedom
Provide good defaults and undo
Help
Provide proactive and in-place hints to guide users
Error Prevention
Help users avoid making mistakes
  • Slips: User has right mental model of system but evidently does wrong thing.
  • Mistakes: User does exactly what they intend to do but leads to error.
Recognition
Make information easy to discover
Minimalism
Provide only necessary information in an elegant way
Flexibility & Efficiency
Make advanced tasks fluid and efficient
Error Recovery
Help user recognise, diagnose and recover
Consistency
Use same interface and language throughout
Mapping
Use familiar metaphors and language
Visibility
Show system status, say what’s happening
