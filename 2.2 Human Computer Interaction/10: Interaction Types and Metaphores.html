Interaction Type: What user is doing when interacting with system e.g. instructing, conversing, manipulating.

Interface type: Kind of interface used to support the mode e.g. speech, menu based,  gestures.

Conceptual Models
Manipulating
Involves dragging, selecting, opening, closing and zooming actions on virtual objects. Includes both physical controllers or air gestures e.g. Kinect. 

Direct Manipulation: Schneiderman (1983) principles:
  1. Continuous representation of objects and actions of interest.
  2. Rapid, reversible, incremental actions with immediate feedback.
  3. Physical actions and button pressing instead of syntax.

Adv/Dis
  • Novices and experienced users can both use rapidly.
  • Reduces memory loss from infrequent users.
  • Immediate feedback on actions.
  • Users feel in control
  • Not all tasks can be described by objects and actions e.g. word replacement in word processor.
  • Moving mouse around green can be slower than using commands.
Instructing
Issuing commands and selecting options, can be done through buttons, menus, command-line…
  • Quick and efficient, making it good for repetitive actions.
Conversing
Interacting with system as if having a conversation. Can features intelligence to personalise conversation e.g. Siri, Google Assistant, Alexa, telephone systems, toys and robots.
  • Users interact with devices in familiar manner.
  • System may not know how to parse given command.
  • Not yet socially acceptable to converse with technology.
Exploring
Involves users moving through virtual or physical environments. Controls may be of any form. It’s purpose can be for entertainment, navigation, visualisation.

Proxemics: Edward Hall (1966) the amount of space that people feel it necessary to set between themselves and others. Has a large impact on how users interact with space.



Choosing best model
  1. Type of task: Is it repetitive, for children, designing, driving etc...
  2. Who is in control: Direct manipulation often gives user full control.
  3. User Requirements
  4. Budget and other constraints


Interface Metaphors: Way of abstracting functionality to users making it easier to use the system. For example, the way we perceive our folder structure is very different to how it is represented in the file system.
    
    Advantages
  • Makes learning new systems easier.
  • Helps users understand underlying conceptual model.
  • Increase diversity of users able to use system.

    Disadvantages
  • Often have to break the rules required for the metaphor.
  • Can restrict range of useful tasks.
  • Not being able to understand system beyond the metaphor.
