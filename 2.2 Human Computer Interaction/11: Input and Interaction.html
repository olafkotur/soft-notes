Input Control
  • Semantic Matching: Pre-defined mapping of button or key to a function.
  • Spatial Matching: Matching to control based on its position.



Types of Input
  1. Fingers - most common
  2. Hands - tilting, gestures
  3. Feet - pedals
  4. Head - pointing
  5. Voice - commands
  6. Body - gestures, proxemics
  7. Face - expressions
  8. Eyes - pointing, interesting as you can command what you want to see unknowingly.

Abstraction - Keyboard Example



Hard vs Soft Controls
  • Hard: Keys, keyboards, mouse, buttons etc..
  • Soft: Click, double-click, drag etc...
  • Easy way to remember is that soft controls are determined by software
  • Need to be careful when distinguishing as all soft relies on hard.
  • Some cases are less effective e.g.  soft keyboards ~45wpm compared to hard ~80wpm.

Three-tier spatial relationship
  1. Control space
  2. Display space
  3. Control-display mapping



Control-display relationships
  • Gain: Scales factory when mapping control to a display: CDgain = Vpointer / Vpointer
  • Latency: Delay between input action and corresponding response. Includes sampling rate, display frame rates, computation, communication.
  • Property sensed and order of control: Position, displacement, force.

Modes: Same user action can have different effect, depending on what mode system is in. Solves problem of supporting more functions than controls. E.g. CAPS LOCK.
  • User can forget which mode system is in.
  • May not reset to default mode after goal is complete.
  • Not noticing system returned to default mode.


Time scale of human actions (Newell 1990)



Other time constraints
  • 0.1 Second: Limit for cause-effect perception
  • 1 Second: Users flow of thought to stay uninterrupted
  • 10 Seconds: Limits for keeping users attention

Keystroke Level Model (KLM)
  • Humans similar physically thus can predict physical reaction times for input.
  • Assume that we always know where we are going in interface.
  • Can group tasks into 3 categories: Physical operators, Mental preparation, System response.

Fitt’s Law: Movement time of a well-rehearsed selection task increases when distance increases and decreases when target size increases.




Steering Law: Adaptation of Fitt’s law, tracking a constrained path takes longer, e.g. choosing an option from a drop down menu. 
MT = a + b * (A/W)
  • A is the tunnel length
  • W is the path width
