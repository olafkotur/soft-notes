Key questions for user study
  • Sampling: Who you select to participate
  • Design: What tasks, under which conditions (context)
  • Measurement: What data you collect and how it will be collected.
  • Analysis: How you analyse, present and report (interpret?) data.

Evaluation Types
  • Formative: Make improvements prior to release, identify and diagnose problems.
  • Summative: Check how well objectives have been met, evaluate against set criteria.

Usability Issues: Problems users encounter toward achieving their goals.
  • Think aloud protocol: Users report what they’re doing/thinking.
  • Listen and observe: Signs of confusion, frustration, surprise.
  • Issue observed with one user may not be an issue with another user.
  • Can create metrics based on how users responded to the interface.
  • Measure certain issues based on severity.
        
    

Measuring Performance
  • Measure how users perform with the interface, can they reach their goals, how long did it take.
  • Users vary on how well they can do a task, important to use a lot of users to base the results on a representable average.
  • Clearly define task success and failure, can represent such data using rates.
  • Measure efficiency based on how long a task took under different conditions, however, efficiency can also be measured whether or not task was completed with minimal effort. 
  • Measure learnability by analysing whether user becomes proficient.



Measuring User Experience
Open Questions: Interview led by un-scripted means, can ask for any unprompted opinion. Often used where user has opportunity to provide more in-depth knowledge. Subjective thus difficult to analyse. 
Closed Questions: Restrict responses using pre-determined means. Easy to analyse and repeat. Often provide constrained answers as users don’t have opportunity to expand on their answers.
Other means
  1. Ranking: Respondent asked to place ordering on items in list.
  2. Likert scales: User judges statement using a numeric scale.
  3. Semantic differential scale: Using pairs of adjectives to describe an experience.
  4. System Usability Scale (SUS): John Brooke 1986, includes 5 positive statements, 5 negative statements (Strongly disagree Strongly agree).
  5. Usefulness, Satisfaction and Ease of Use (USE)
  6. NASA TLX
  7. Attrakdiff2 

Practical Guidelines
  • Demographics
  • Concise
  • Relevance
  • Precision
  • Leading questions
  • Avoid ‘and'
  • Avoid negative questions
  • Avoid abbreviations
