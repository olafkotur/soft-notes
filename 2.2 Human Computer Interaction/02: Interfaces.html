Computers
  • Functional: Small set of arithmetic operations, use functions that take input to produce output using algorithms. Thus governed by all of these aspects.
  • Deterministic: Algorithms and instructions will always be same, unpredictability will come from input.

Input and Output
  • Input: Any kind of data user provides, provides controls through which users can express input in valid structures.
  • Low-level: Keystrokes, mouse position
  • High-level: Clicks, taps, gestures
  • Output: Result produces when input data is used in algorithms and functions. They can be presented in many forms e.g. list of results, images, related lists.

WIMP Paradigm
Windows
Provide visual access to content that is larger than size of screen. Can navigate using directional scrolling e.g. swipe gestures, scroll bars, head movement in VR. User can manage window by changing size, zooming, full-screen, tiling more than one window.
Icons
Map actions to pointing device which we would otherwise need commands. We can do things such as click to launch, drag and drop, open menus.
Menus
List of useful functions that user can call to perform action, can view this in list of available commands that can be selected by pointing. We can use widgets to provide input data such as sliders, checkboxes and text boxes.
Pointer
Direct manipulation was proposed by Ben Schneiderman in 1983, characteristics include:
  • Continuous representation of object of interest.
  • Physical actions instead of complex syntax.
  • Continuous feedback and reversible, incremental actions.
  • Rapid learning.


Model-View-Controller (MVC)
  • Model stores data user interface is presenting e.g. scroll bar’s data is window position over the content it displays.
  • View visualises data e.g. drawing at which position of content user is on.
  • Controller manages state machine, transitions to states in response to events, executes commands and updates model e.g. handling clicks of scroll bar and updating window position of model.

View Hierarchies
  • All user interfaces are structured as view hierarchies (view trees) e.g. picking date.
  • Output: User interfaces change their output by mutating the view tree. Re-draw algorithm updates affected areas.
  • Input: Views have input handlers, view tree controls propagation of input events. 
  • Layout: Algorithm traverses the tree and calculates positions and sizes of views.


Event handlers
  • An event source generates stream of discrete events e.g. mouse movement coordinates or keystrokes such as keyup and keydown with a key code. 
  • Views register interest (subscribe) providing function to be called when new event occurs e.g. may only subscribe to mouse events that occurs within view’s boundary.
