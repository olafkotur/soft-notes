Delay: Measure of time taken for packet to travel across the network, measured in fraction of seconds (usually ms).

Types of Delay: Put together they are known as Nodal Delay
  1. Processing Delay
  2. Queuing Delay
  3. Tranmission Delay
  4. Propagation Delay



Processing Delay
  • Time taken for device to examine packet’s header and decide where to direct it.
  • Includes check of bit level errors caused during transmission. 
  • Varies depending on how busy the device is.
Queuing Delay
  • When packet is processed, it will enter queue and not leave until reaching the head of the queue. 
  • Length of queue depends on congestion level of router.
  • Queue only occurs if packet arrival rate to link exceeds output link capacity. 
Tranmission Delay
  • Store-and-Forward: Entire packet must be received before it is forwarded.
  • Amount of time required to push all of packet’s bits into the link.
  • packetLength / linkBandwidth = Delay
Propagation Delay
  • Time necessary to get from one device to another across a transmission medium.
  • linkLength / speed(~2x108 m/sec) = Delay

End-to-End Delay: Takes into account all of the Nodal Delays from one host to another, includes numerous nodes. 

Round Trip Time: End-to-End in both directions, i.e. one to host to another, and then back again. Doesn’t necessarily always take the same path/route.

Measuring Delay - Traceroute: Provides delay measurement from source to router along end-end Internet path towards destination.
  1. Sends three packets that will reach device on path towards destination
  2. Device will return packets to sender
  3. Sender times interval between transmission and reply
> traceroute google.co.uk




Throughput: Rate at which bits transferred between sender and receiver. Restricted by bottleneck, can use protocols to throttle and reduce bottleneck but lowering the raters.
  • Instantaneous: Rate at given point in time
  • Average: Rate over longer period of time
  • Peak: Highest rate



Internet Scenario



Goodput: Measures throughput at the highest level to get an ‘honest’ measurement. Can be measured at any point of entry e.g. Internet speed test will test the speed at which it enters computer which can be affected by a Network Card or other factors.

Measuring Throughput - iperf: Standardised throughput measurements, runs on client-server model.
  1. Host request data, other serves it
  2. We know the size and long it takes to receive the data, thus calculate.




Packet Loss: Occurs when node drops / discards packet (doesn’t reach destination).
  • Reduces useable throughput of device as number of useful bits sent is less.
  • After a period of time, a retransmission request is sent. Causes latency issues as packet has to travel full journey again.
  • When packets arrive at rate greater than max supported throughput a queue starts to fill
  • We have control over which packets we wanna drop e.g. FIFO, Random, Priority

Causes
  1. When queue is full packets are discarded
  2. Lost in the physical medium especially with unguided radio technologies
  3. Hard/Software failures causing errors and corruption - failed checksum
  4. DDoS- bombard with packets so it can’t receive

Measuring Loss
  • Creates and sends messages to a specific host.
  • Measures end to end delay
  • Keeps track of how many messages sent and receiving thus packet loss.
  • Can be done via ping
