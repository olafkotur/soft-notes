Dijkstra Algorithm: Finds shortest path from source to all other destinations.
  • Find known nearest neighbour
  • See if we can reach others from that neighbour by shorter route than previously
  • Using nearest ensure that there can be no shorter path

Implementation
  • Each node periodically generates Link State Packet
  • ID of node generating the LSP
  • List of direct neighbours and costs
  • Sequence number (64 bit) and TTL.
  • Neighbours are flooded
OSPF
IS-IS
  • Favoured by companies and datacenters.
  • More optional features.
  • Built on top of IPv4 but need OSPFv3 for IPv6.
  • Favoured by ISPs
  • Less chatty than OSPF
  • Less network overhead
  • Supports more devices
  • Not tied to IP thus works with v4 and v6

ASN: Globally identifies a system allowing them to share routing information with their neighbours.
  • Limited number of ASN as they are 16 bit values and also 64512 - 65535 is reserved.
  • Currently there are around 40k of ASN e.g. Google, Facebook, Lancaster University

Intra/Inter AS Routing
  • Routing among routers in same AS must run same routing protocols
  • Border routers have links with other routers in other ASes
  • All inter-domain routers use BGP routing protocol
  • Inter perform both inter and intra domain routing.




Border Gateway Protocol (BGP):  Manages how packets are routes across network through the exchange of routing and reachability information between edge routers.
  • Driven by economics - these are business entities
  • Has power to route information via a different path.

Peering: Becoming buddies with another provider, cooperative but competitive
  • Reduces upstream costs
  • Improves end to end performance
  • May be the only way to connect to parts of the internet
  • Peers are often competition thus often require periodic renegotiation.
