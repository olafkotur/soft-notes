Network Structure
  1. Network Edge: Hosts, clients and servers
  2. Access networks, physical media: Wired and wireless links
  3. Network Core: Interconnected routers

Edge Networking 

Access Network
Digital Subscriber Line (DSL)
  • Use existing telephone line to central office DSLAM - takes bundle of wires and separates data and voice.
  • Data over DSL phone line goes to Internet
  • Voice over DSL phone line goes to telephone net
  • Typical speeds: < 2.5Mbps UP | < 24Mbps DOWN

      

Cable Network
  • Uses TV network to get internet - allows both TV and Internet to operate.
  • Frequency Division Multiplexing: Different channels transmitted in different frequency bands - allows multiple users.

         

Home Network
  • Router is attached to a modem (often same device).
  • Has predefined functions including security which allow multiple devices to connect without having unique addresses.
  • Users may often compete as they are on the same network thus lower speeds. 

     

Enterprise Access Networks (Ethernet)
  • Used in big areas such as companies and universities. 
  • Speeds can range range from 10Mbps to 10Gbps.
  • End systems usually connect into Ethernet Switch.

      

Wireless Access Networks
  • Shared wireless access network connects end system to router via base station (access point).
  • Wireless LANs - within a building up to 100ft.
  • Wide-area wireless access - provided by telco (celluar) operator and can give access at even 10km i.e. 3G, 4G, LTE

Host: Sends packets of data
  1. Takes application message, usually fixed number of bits.
  2. Breaks into smaller chunks (packets) containing a number of bits.
  3. Transmits packet into access network at a transmission rate depending on many factors including characteriscs of medium. 

Guided Media: Signals propagate in solid media: copper, fiber, coax.
Unguided Media: Signals propagate freely: radio

Physical Media
Twisted Pair
Two insulated copper wires.
Coax, Fiber
Coaxial: Two concentric copper conductors which are bidirectional allowing multiple channels.

Fiber: Glass fibre carrying light pulses, high speed operation with low error rate as it is immune to electromagnetic noise.
Radio
Signals carried in electromagnetic spectrum, allows bidirectional signals but have limitations: 
  1. Reflection
  2. Obstruction by objects
  3. Interference i.e. other wires, devices etc


Core Networking

Packet Switching: Hosts break application-layer messages into packets.
  • If arrival rate to link exceeds transmission rate of link for a period of time:
    • Packets will queue, wait to be transmitted on link.
    • Packets can be dropped off memory if buffer fills up.

Store and Forward: Entire packet must arrive at router before it can be transmitted on next link.

Key Network-core Functions
Routing
Determines source destination route taken by packets
Forwarding
Moves packets from router’s input to appropriate router output.

Circuit Switching: End-to-end resources allocated.
  • Reserved for ‘call’ between source and destination. 
  • Resources are dedicated, no sharing allows guaranteed performance.
  • Segment is idle if not used by call. 
  • Used in traditional telephone networks.

FDM vs TDM


Packet Switching vs Circuit Switching
  • Packet is very good for sending data amongst lots of users as the resources are shared and little is idle thus high efficiency. 
  • However, Circuit can reduce packet delay and loss which guarantees reliable data transfer and congestion control. 


Internet Exchange Point (IXP): Neutral entities that will connect multiple ISPs together allowing interconnection.



Transit ISPs: Dedicated to carrying other ISPs traffic. They charge for providing this service. Usually levied on the origin network.

Peering: ISP can peer directly between each other, in a mutually beneficial cost-free agreement; must be worthwhile for both parties.

Content Provider Networks: Companies may run their own network to bring services and content to close end users e.g. Google, Microsoft, Akamai.
