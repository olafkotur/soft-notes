Reliability: Degree to which result of a measurement, calculation or specification can be depended on to be accurate.
  1. We want to ensure remote operation has been successful.
  2. We want to guarantee the ordering of operations on remote client.
  3. We want to know that operation has been lost or failed.

TCP Characteristics
  1. Point to point: One sender, one receiver
  2. Pipelined: TCP congestion and flow controlled.
  3. Reliable: In order byte stream with no message boundaries.
  4. Connection-orientated: Handshaking initiates sender, receiver state before data exchange.
  5. Full duplex data: Bi-directional data flow in same connection.
  6. Flow controlled: Sender will not overwhelm receiver




Sequence numbers: Byte stream of number of first byte in segment data.
  • Acknowledgements (ACKs): Sequence number of next byte expected from other side.

Timeout, TTL and RTT
  • Need to be careful that TTL is not too short as it may cause premature retransmissions.
  • If TTL too long then there is a slow reaction to segment loss.
  • Able to estimate the RTT by using an exponential weighted moving average.
  • Timeout Interval: Estimated RTT plus safety margin
  • If there is a large variation in estimated RTT then the safety margin is larger.


Reliable Data Transfer (RDT)
  • TCP creates RDT service on top of IP’s unreliable service.
  • Retransmissions are triggered by timeout events and duplicate ACKs.
  • If sender receives 3 ACKs for same data, resend unasked segment with smallest sequence number



Flow Control: Receiver controls sender that it won’t overflow receiver’s buffer by transmitting too much, too fast.
  • Receiver advertises free buffer space by including rwnd value in TCP header.
  • Guarantees receiver buffer will not overflow.


Connection Management (handshake part)
  • 2-way Handshake: Receiver sends request, sender accepts. But possible failures exist.

  • 3-way Handshake: Receiver sends a SYN data packet to see if server has open connections. Server will respond with a SYN response packet where the receiver will then send out an ACK. When this is completed, the connection will be made.

  • Closing connection: Client and server each close connection separately.
