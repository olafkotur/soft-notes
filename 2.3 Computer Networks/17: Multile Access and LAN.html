Random Access MAC Goals
  • Share the medium: If two hosts are sending at same time then it will collide,
    However, if no hosts are sending then channel is idle. Ideally 1 user is sending at all times.
  • High utilisation: TDMA is low utilisation.
  • Simple, distributed algorithm: Allow multiple hosts that can’t directly coordinate to talk in a non-fancy simple way.
Some Protocols
ALOHA
If you receive transmission from another station while transmitting there has been a collision thus just transmit later.
Slotted ALOHA
Start transmissions only at fixed time slots, significantly fewer collisions than ALOHA. If there is more than 1 frame then don’t receive.
Carrier Sense Multiple Access (CSMA)
Start transmission only if the channel is idle. Listen to channel if signal is present before transmission. 
CSMA / Collision Detection (CSMA / CD)
Stop ongoing transmission if collision is detected - there’s no point sending if a collision is detected.

ALOHA vs TDMA
  • Using TDMA each host must wait for their turn.
  • Thus delay is proportional to number of hosts.
  • With ALOHA each host sends immediately thus much lower delay but also lower utilisation
  • Slotted ALOHA has twice the throughput than normal ALOHA.
  • However, hosts must have synchronised clocks putting best possible throughput at 37%.


Exponential Back-off: After collision is detected the algorithm increases delay time before retransmission.
  • If more than one collision occurs the length of delay is increased exponentially.





MAC Address: 48 bits for most LANs e.g. 1A-2F-BB-76-09-AD
  • Used locally to get frame from one interface to another that is physically connected.
  • Allocation administered by IEEE
  • Manufacturer buys portion of MAC address space to ensure unique.


Address Resolution Protocol (ARP): Used to link IP addresses with MAC addresses using a table mapping.
  • Each IP node has an ARP table i.e. <ip_address; mac_address, ttl>
  • TTL is usually 20 minutes for address mapping.
  • Nodes able to create their ARP tables without network administration - plug and play.





Ethernet: Active switches are in the centre where each “spoke” runs an Ethernet protocol.
  • Doing so nodes do not collide with each other
  • Different speeds of ethernet exist (2Mbps, 10Mbps, 100Mbps, 1Gbps, 10Gbps)
  • Different physical layer media e.g. fiber, cable
           


802.3 Ethernet
  • Preamble: Syncs clocks
  • SF: Start frame
  • Includes source and destination MAC addresses
  • The minimum packet length is 64 bytes hence the padding


Bridges: Connect multiple IEE LANS at layer 2
  • Reduce the collision domain
  • Having things in between we reduce the chance of collision - kinda like checkpoints.
  • Host do not know if there are bridges between sending - transparent and abstract
  • This is good because we don’t have to do any extra configuration when changing shit
  • Each bridge has a forwarding table with the MAC address, port and age.


Learning Address: Whenever we send a frame we are able to note down the address and port it came from
  • Means we don’t have to manually enter things - as that is time consuming and error prone.
  • Also used in interconnecting switches - this means that hosts don’t need to be configured to know which route to take
