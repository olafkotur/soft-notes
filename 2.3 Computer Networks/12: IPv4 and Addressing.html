IPv4: 32-bit addresses written in dotted notation where each number is 8 bits.

Challenges
  • Routers will need to know every possible IP for the next hop.
  • Using 32-bits that’s 2^32 possibilities (4,294,967, 296) - too slow.
  • Thus routing table is required at all routers.
  • Need to have a structure to the IP i.e. a prefix, network and then the host itself.

Classes of IP addresses


Classless Inter-Domain Routing (CIDR): Flexible Dickson between network and host addresses.
  • Removes need for IP classes.
  • Network prefix can be any size. 
  • Mask is a 32-bit number that determines network part and the host part.
  • Offers a better tradeoff between size of routing table and efficient use of IP address space.


Subnet: Device interfaces with same network part of IP address i.e. can reach each other without another router.


IP Functionality: Solved by an IP Datagram.
  1. Delivering Packet: Where is the packet going and which protocol will process packet on host?
  2. Network Handling: How should packet be forwarded (priority) and where does header and packet end?
  3. Problem Coping: Is the packet corrupted, fragmented or broken in any other way?


Fragmentation: Split datagrams into pieces when maximum transmission (MTU) unit is reduced.
  • Can’t always predict the MTU for the entire path thus might hit a reduced unit at some paths.
  • Reassembly happens at the end host as fragments can travel across different paths.
  • Causes inefficient use of resources as you have to split it and then put it back together.
  • Can lose fragments along the paths thus degrading performance - loss of fragment means whole datagram retransmit.


IPv6: Introduced in 1998…
  • 8 groups of 16-bit values, separated by :
  • Leading zeroes in each group may be omitted using ::
  • Switching to IPv6 is a whole-internet upgrade - all routers and all hosts.
  • IPv4 is the glue for all protocols and physical links thus really fucking hard to change this to IPv6

IPv6 Consequences
  • We use blacklists in IPv4 to block certain spammers and bots, due to limited space it's hard for them to get new IPs.
  • Using IPv6 the address space is so goddamn big its gonna be impossible to use blacklists.
  • Hard to manage subnets allocation within networks.
  • Transition is gonna be hard as fuck.
