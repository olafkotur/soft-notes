Routing Protocols
Per-Router control plane
Individual routing algo components in each and every router interact with each other in control plane to compute forwarding tables.
Logically centralised control plane
Distinct controller interacts with local control agents in routers to compute forwarding tables.
Routing Algorithms
Calculate the shortest path whilst taking into account the load across the network and ensuring the lowest cost of use.
  • Constructed using graphs and nodes
  • Each node only knows the cost of its neighbours.
Intra-domain Routing Protocols
  1. Distance Vector: Routing Information Protocol (RIP), route rs periodically exchange reachability information with neighbours.
  2. Link State: Open Shortest Path First (OSPF), each network periodically floods immediate reachability information to all other routers - Dijkstra
  3. Routes will only change if the algorithms determine that a shorter route is present.

Routing Information Protocol (RIP)
  • Uses distance vector i.e. graph costs
  • Updates sent every 30 seconds
  • No auth for updates
