Hourglass Model: Few network protocols are the glue that allow inter operation



Transport services and protocols
  • TCP and UDP
  • Provide logical communication between app processes running on different hosts
  • Transport protocols run in end systems
    • Send side - breaks app messages into segments and sends into network
    • Receiver side - reassembles segments into messages, passes to app layer

Transport layer service
  • Connection oriented: Connection set-up, data transfer, disconnect.
  • Connectionless: Transfer of isolated units
  • Logical communication between processes

Transport Protocol Data Unit (TPDU)



Multiplexing & Demultiplexing
  • Combines multiple signals into one in order to share a resource.
  • The sender handles data from multiple sockets and adds a transport header
  • The receiver uses header info to deliver received segments to correct socket
  • Host receives IP datagrams where each one has a destination port to send to appropriate socket.
  • Each datagram has source IP and destination IP.
      

Connectionless demultiplexing



Connection-orientated demultiplexing
  • Host may support simultaneous TCP sockets due to each socket having own 4-tuple.
  • Web servers have different sockets for each connecting client - may be threaded
  • Uses 4 values for demux
    • Source IP address
    • Source port number
    • Destination IP address
    • Destination port number





User Datagram Protocol (UDP)
  • Best used for low tolerance connections such as multimedia applications rather than video calling.
  • Reliability is added on the application layer thus it is app specific error recovery.
  • UDP header and data minimum 8 bytes, maximum is (2^16 - UDP header) in an IPv4 network layer
      

Why UDP?
  1. No connection establishment reducing delay
  2. No connection state at either sender or receiver thus simple
  3. Small header size
  4. No congestion control as UDP can just blast away


UDP Checksum: Goal is to detect errors in a transmitted segment. Done by seeing any flipped bits in a data stream.

Receiver
  • Treat segment contents as sequence of 16 bit ints.
  • Performs one’s comp of segment contents.
  • Sender puts checksum value into UDP
  • Compute checksum of received segment.
  • Check if computed checksum equals sent.
  • Decide what to do with the data.

How to calculate it
  1. Add numbers together
  2. If there’s an overflow then take the overflow and add it to the lowest sig bit
  3. Convert to one’s complement
