Network Layer Implementation
  • Network Interface Card (NIC) or on a chip
  • Ethernet Card or Ethernet chipset
  • Implements link on a physical layer
  • Encapsulates datagram in a frame and adds error checking bits i.e. rat, flow control...
  • Looks for errors i.e. rat, flow control… and extract datagram passing it to upper layer.

Dealing with noise
  • Interference from electrical cables
  • Cross-talk from radio transmissions and microwaves
  • Solar storms



Error Detection using Redundancy
Naive Error Detection
  • Send 2 copies of each frame.
  • High overhead
  • Poor protection against errors - twice the chance for bit errors.
Cyclic Redundancy Check (CRC)
  • Uses field theory to compute semi-unique value for given message.
  • Fixed size overhead per frame - usually 32 bits
  • Bits exactly divisible by N (agreed) - if result is non-zero then an error is detected.
Parity Bits
  • Add extra bits to keep the number of 1s even or odd.
  • Add either 1 or 0 to even the bits in a string of bits.
  • If this is not the same on the other side then something must have gone wrong.
  • E.g. 7-bit ASCII + 1 parity bit
  • 2D Parity bits, 1 for each row and 1 for each column

Media Access
  • Simultaneous transmissions in medium cause collisions - destroys the data.
  • Media Access Control (MAC) protocols state rules on how to share medium
  • Strategies for detecting, avoiding and recovering from collisions.
MAC - Strategies
Channel Partitioning
  • Divide resource into small pieces and allocate each piece to one host
  • Just like TDMA and FDMA
Taking Turns
  • Tightly coordinate shared access to avoid collisions
  • Just like token ring networks
Contestation
  • Allow collusions but use strategies to recover
  • Used in Ethernet and Wifi
  • Random Access MAC Protocol specifies how to detect and recover e.g. Slotted ALOHA, ALOHA, CSMA
