Control models: Control flow between sub-systems. Distinct from system structuring models which are static representations.
  • Centralised control: One sub-system has overall responsibility.
  • Event-based-control: Each sub-system can respond to externally generated events from other sub-systems or the system’s environment.

Centralised control
  • Call-return model: Top-down approach where control starts at top of subroutine hierarchy and moves downwards. Applies to sequential systems.
  • Manager model: One systems controls stopping, starting and coordination of other system processes, applies to concurrent systems but can be implemented in sequential systems using case statements.




Event-driven systems: Driven by externally generated events where timing of event is out of control of sub-system which process event.

Broadcast model
Interrupt-driven models
Event is broadcast to all sub-systems, any sub-system can handle event may do so. 

Systems can register interests in events, then control is transferred (publisher-subscriber pattern).

Sub-systems decide on events of interest to them, however, sub-systems don’t know if or when event will be handled.




Used in real-time systems where interrupts detected by interrupt handler and passed to some other component for processing (fast response, but complex to code and validate).

Known number of interrupt type is defined, each type is associated with memory location where handler’s address is stored then hardware switch causes transfer to handler.



Examples: Anti-lock braking system, airbags, train protection warning, auto shutdowns.
