Non-correctness
  • Sequential program will always perform deterministically if same given input
  • Concurrent programming not always the case thus leading to special languages like Eiffel, CSP and SR which to try to aid correctness.

Deadlock: Two ore more threads blocked forever waiting for each other.


Livelock: Occurs when deadlock detection repeatedly triggered, leads to process starving in situations where there is always a runnable process with higher priority.
  • Starvation: Process is unable to obtain regular access to share resources- problematic for greedy threads due to bad algorithm design.
