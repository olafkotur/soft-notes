Concurrency: How multiple, independently controlled processes behave when running and interacting with each other.
  • No matter how many threads there are, and what order the program is executed in, the output should always be the same.
  • Does it sequentially but perceived as concurrent due to speed of execution.
  • Not classes as parallel as concurrent is possible to have more than one, or none running at the same time.

Concurrency Components



Computer Systems Concurrency
  • Scheduler: Method for how work is assigned to resources to complete work.
  • Thread: Smallest sequence of instruction managed independently by scheduler.
  • There are multiple actors (processes) inside an OS, there are one or more threads in single process, there can often share resources like RAM. 


Three Central Concepts
Processor
Hardware device that executes machine instructions.
Program
Instruction sequence defining potential execution path passive description of what you would like to happen stored on disk.
Process
Active system entity executing associate program or; Program in execution on processor resides in primary memory, remove on reboot. Process can run multiple versions of same program. 

Disturbed sequential execution
  • Interrupt processing, handling external events such as keyboard of mouse clicks.
  • Context switch: Execution jumps to another part of memory.

Applications
  • GUI based app listens for key presses, mouse clicks.
  • Web server handles multiple requests at same time.
  • Sensor monitoring and collection.

Scheduler (adversarial)
  • Can guarantee no starvation i.e. it will run, one day.
  • Cannot guarantee fairness i.e. processors will run the same amount of time.
  • Cannot guarantee timeliness i.e. complete process in desirable time.
