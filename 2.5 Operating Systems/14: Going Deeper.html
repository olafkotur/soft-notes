Scheduling Criteria
  • CPU Utilisation: Percentage of time CPU running application code
  • Throughput: Number of processes completed per unit time
  • Turnaround time: Time between process submission and completion
  • Waiting time: Sum of waiting in Ready Queue
  • Response time: Time between submission and start of first output/interaction.

Average Waiting Times: Dispatch time - Arrival time



Average Turnaround Time: Completion - Arrival Time



Time Quantum: Amount of time share given to each process, interrupting the job if it is not completed by then and moving onto the next process.

Burst Time: Processing time for a specific process, may have to go back on queue if not completed in a given time frame. 


Round Robin Scheduling Sequence
  • Build the request queue for every time quantum
  • Example, P1, P2 and P3 arrive at 0, 1 and 3 where the TQ = 3 thus put them all onto the queue.
  • Some process have to be put back onto the queue e.g. P1 after P3 as their burst time exceeds the time quantum.



Variable Sized Allocation Issues



Bitmap: Keep track of allocated memory using an allocation map. If the memory is free then note this with 0s otherwise 1s.
  • Compact and fast to search for free entries
  • Uses very little space to note the memory management thus little overhead
  • If a process needs an ’n’ size of memory we must keep searching until a space big enough fits the request.
