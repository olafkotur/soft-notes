Shared-Memory Concurrency: Synchronised statement that locks, doesn’t allow for concurrent programming models e.g. distributed systems.

Mutual Exclusion: Only one thread can run within a critical section at a time.

Mutex: An object that ensures threads take turns in accessing a shared resource.

Message Passing: Process can communicate by sending and receiving messages. Form of inter-process communication (IPC). Includes communication between:
  • Different threads within a process
  • Different processes running on same server
  • Different processes on separate servers

MP Message
  • Two interacting processes located on same processor, channel is processor’s local memory.
  • Two interacting processes allocated in different processors, channel mapped to physical communication medium.
  • Message transfers number of data items of certain type from memory of one process to memory of another.
Two abstract primitives
Send( )
Send message to a specific channel
Receive( )
Receive message from a specific channel

Synchronous MP
  • Require sender and receiver to wait for each other
  • Send and receive is blocked until message is sent to channel
  • Sending process will not return until message is out of source memory



Asynchronous MP
  • Don’t wait, carry on their own computation while message is transferred
  • Send process is not blocked while receive is blocked until message is sent to channel
  • Process sends message and continue execution without waiting for message to be received
  • Some form of buffer is deployed for async message passing




Synchronous Problems
Asynchronous Problems
Reduced Concurrency: Whenever two processes communicate, at least one of them will have to block.
Acknowledgement: No current state of sending process, no way of knowing if received unless send replies.
Client-server interactions: When client releases resource, usually no reason for waiting until server received message.
Failure: Difficult failure detection, finite buffer space if too many messages are sent (DDOS), will crash.

Mutual Exclusion with MP
  • Dedicated process supervises critical section and executes on behalf of requesting process.
  • Process wishing to access critical section sends request to server.
  • Server takes requests serially and ensures mutual exclusion.
