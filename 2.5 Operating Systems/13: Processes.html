Process: Loaded instance of a program, defined by its context which includes:
  • Open files
  • Contents of CPU registers
  • Memory map (segments, pages)
  • On-going communication e.g. signals

Basic Process Control Block (PCB)




Batch Systems: They are non pre-emptive, they run until blocked or complete. Follows first come first served policy. Use moving average to predict next expected run time to do shortest job first.

Characterising Processes
  • CPU Bound: Process spends most of its time processing data
  • I/O Bound: Process spends most of its time waiting for data

Fair Share Scheduling
  • Unfair to split resources equally as it should take account of process hierarchy (some processes may have more child processes than others).
  • Parents responsible for children thus children share the parent’s time and resource allocation.

Scheduling Overhead: Cost to every process switch.
  1. Interrupt Latency
  2. Save current context 
  3. Select next process
  4. Load new context



Real-Time Systems
  • Soft Real-Time: Deadlines may be broken (multimedia)
  • Hard Real-Time: Deadlines are critical (control systems)
  • Scheduling is driven by events either periodic (predictable) or aperiodic. Must ensure work can be scheduled thus can never take too much work or use greater than 100% utilisation.

Rate Monotonic: Static scheduling policy - processes have fixed schedule based on priority. Must also schedule for worst case.

Earliest Deadline First: Dynamic scheduling scheme, each process has sequence of deadlines, process scheduled with closest deadline.



Interactive Systems: Cooperative system where tasks and processes actively yield control to scheduler (explicit calls to scheduler coded within task or process). Scheduler forcibly takes control at regular intervals. 

Pre-Emptive Scheduling: Scheduler forces context switch at defined interval, Quantum determines how long processes run, defined as long (fewer switches) and short (more switches).
  • Round Robin Scheduling: Perform a task on each process and then return to the first process. Time slices  assigned to each process in equal portions and in circular order, handling all processes without priority.
  • Priority Scheduling: High processes selected over lowers. Can have multiple priority queues by having Round Robins within each priority level.
           




Priority Inversion: When a high priority process needs resource from low priority process the priority is briefly inverted so that the resource is accessible.

Interrupted interrupts
  • Interrupts themselves can be interrupted and so on.
  • Creates a nest of interrupts which complicates code and timing but is sometimes essential.
  • This is done via Interrupt Request Hardware Lines (IRQL)


Deferred Procedure Calls (DPCs): High priority tasks are able to defer lower but also required tasks to be scheduled for a later time.
  • Interrupted Service Routine (ISR) is called that does the real work
  • Clears interrupt flags etc.


Schedulers and the 
Dispatcher: Code that actually (re-)starts process
Long-term Schedules
Given new process, performs admission test i.e. should process be allowed to run and added to queue?
Medium Term / High-level Scheduler
Handles course grained issues including swapping and sets process priorities and CPU affiliation.
Short Term / Low-level Scheduler
Called every time slice and decides which process should be picked from run queues.

Process States: Pretty simple, a process is either interruptible or uninterruptible.
