Establish ATM functionality
Bank owns number of branches countrywide, each branch is responsible  for a number of Automated Teller Machines (ATMs) in its locality.
  • Each ATM contains an embedded software system to drive machine hardware and to communicate with the bank’s customer account database.
  • The system accepts customers requests and provides the following services:
    1. Dispenses cash, displays balance
    2. Displays and prints a short statement
    3. Allows PIN change
  • All ATM users issued with cash-card and PIN that they must use to access ATM services.
  • ATM verifies cards locally but communicates with remote customers database to verify PINs, verify transactions and update accounts.
  • ATM users access its services through menus-based interface that allows them to selected desired services.

UML Extensibility: UML designed to be extended in controlled ways, has three extensibility mechanisms.
  1. Stereotypes: Allow for new entities to be derived from existing ones, e.g. exceptions are classes whose instances are thrown.
  2. Tagged values: Allow for additional information, in form of properties and values, to be associated with an entity e.g. author and version info can be added to class.
  3. Constraints: New rules added to entities or existing rules to be overridden. OCL (Object Constraints Language) describes rules that apply to UML models.
