Perfomance Implications
  • Expensive: Takes a lot of processing and memory in order to grow the stack each time the method is executed.
  • Overheads: Java recursion has a larger dynamic RAM footprint which grows linearly with the number of calls - known as a stack explosion.

Types of Recursion
Head
First statement in a method is recursive
Middle / Multi
One or more arbitrary statements in method is recursive
Tail
Final statement in method is recursive
  • Can be optimised by compiler to prevent stack explosion
  • Since its at the end it can just rewrite the stack
  • Happens in most languages
