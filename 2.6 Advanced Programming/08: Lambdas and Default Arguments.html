Lambdas: Arrow notations in ES6 that uses the context of a parent ’this’ instead of creating a new context, developed for three major uses:
  1. Searching - easily implements a comparison operation with little overhead i.e.  [1, 2, 3, 4].filter(a => a > 3); 
  2. Sorting - Similar concept to searching i.e. [1, 5, 3, 6, -4].sort((a, b) => a - b); 
  3. Currying - Return a function rather than performing an operation
    1. Allows partial evaluation - you don’t need to know everything before you start operation
    2. Allows load distribution
    3. Opens interesting parallel and out of order processing options
    4. However, easily misunderstood and can’t always know what is being returned.

Benefits of sorting with Lambda
let data = [
    { x:4, y:10 },
    { x: -4, y:17 },
    { x:3, y:3 }
];

let sortedByX = data.sort((a, b) => a.x - b.x);
let sortedByY = data.sort((a, b) => a.y - b.y);
