Generics: Generic types are defined as classes or interfaces
  • The type is not determined until the first instance is assigned to it.
  • Can fill it with a wildcard type which evaluates to anything <?>
// Generic class
public class LinkedList<E> {
    ...
}

// Specific usage
public static void main(String[] args) {
    LinkedList<Person> library new LinkedList<Person>();
}



Benefits
  1. Extensibility with compile time safety - once created the class is bound to the type thus safe
Limitations
  1. Runtime Type Erasure - meta data of generic types is non-existent thus LinkedList<Video> can’t be distinguished from LinkedList<Person>
  1. Implications of Type Erasure - no guarantee that generic types that are proven type safe at compile time will remain so at runtime.
  2. Class local variables aren’t - generic uses the same class for each generic class you write. Thus any local items defined in a generic class are shared across all versions of a class.

Iterators
public interface Iterator<E> {
    boolean hasNext();    // True if there are more items in the collection
    E next();             // Returns next element in collection
    void remove();        // Removes last element returned by next()
}

// Usage
public class University {
    public void doSomething(Set<Person> staff) {
        Iterator<Person> iter = staff.iterator();
    
        // Method uno
        while (iter.hasNext()) {
            System.out.println(iter.next());
        }    

        // Method duo
        for (Person p: staff) {
            System.out.println(p);
        }
    }
}
