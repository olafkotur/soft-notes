Inheritance: When a class is extended, the instance variables and methods defined in that class implicitly become part of the new class.
  • Provides mechanism to promote reuse and extensibility
  • Code written by one programmer and delivered as a class can then be extended by another to suit needs of the task at hand without access to source code.
  • Legit the backbone of libraries.



Extending Classes: Takes an existing class and adds to it without needing to view and edit the source code.
  • Gives access to all variables in that class, this class is known as a base / super / parent. New class is known as a subclass.
  • We can only ever add functionality not ever remove it.


Constructor: Provide a way to initialise objects when they are created.
  • Super( ) calls the constructor method in the parent class - this must be the first statement in your constructor.
  • This is exactly the same as using the new( ) function.

Method Overloading: We can have more than one method with the same name, in the same class as long as they have a different parameters lists
  • Different to method over riding.


Polymorphism: Any instance of a class which inherits from another can be also treated as an instance of that class.
  • And object reference of one type can refer to an object of a different type
class Apples {
    public static void main(String[] args) {
        Food bucky[] = new Food[2];
        bucky[0] = new Potpie();
        bucky[1] = new Tuna();

        // Can later use this to control all sub classes
        for(int i=0; i<2; i++) {
            bucky[i].eat();
        }
    }
}


Casting: Convert primitive variable from one type to another e.g. int to float
int yolo = 10;
float pengting = (float) yolo;


Abstract: Classes that can be extended but not instantiated
  • Can do the same to methods but with a slightly different effect
  • Any subclass that extends a class containing an abstract method must override that method 
  • This enables programmes to guarantee implementation of a specific method


Interface: Considered similar to an abstract class but without any functionality
  • Canny have any instance variables
  • Canny have any method implementation
  • Provides solution to limitations of single inheritance languages without the complexity of multiple inheritance.
  • Allows you to define what other programmers must do in order to interface with your code.
// Custom interface
interface UniversityLife {
    public void sleep();
    public void drinkBeer();
    public void work();
}

// Enforcing it in another class
class Undergrad implements UniversityLife {
    int drunkness = 0;
    
    public void drinkBeet() {
        drunkness++;
    }
    ...    
    
}
