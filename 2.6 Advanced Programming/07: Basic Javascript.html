Nuances
  • Variable scoping can be more complex than expected.
  • Equality and comparison operators have useful quirks. 
  • Be careful with floating point numbers - 
  • Concatenation vs Addition…
Variables
var
Doesn’t respect block scoping, this is only the case in non-strict environment. This would mean that variables get treated as global variables. Using this is usually discouraged.
let
Respects the block scoping rules and they are mutable.
const
Variables are immutable, enforced by the interpreter. Can be extremely useful to ensure that data can’t be changed by accident. However, can still add to const arrays… For full immutability, use object.freeze();
Equality and Comparators == != === !==
  • Non-Strict Comparison: Each side of operator converted to type that allows conversion then evaluated in place.
  • Strict Comparison: Evaluated without any type conversion, behaves as you’d expect.

Autoboxing
  • The only primitive types in JS are numbers and strings.
  • Whenever interpreter needs to do conversion, it internally ‘boxes’ primitives with their object-form.
