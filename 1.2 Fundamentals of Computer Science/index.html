DS: Dynamic storage and memory management

Heap

  • Contains linked lists of used and free cells (blocks).
  • New cells created using malloc() and redundant cells should be released using free().
  • Memory blocks can be different in sizes e.g.

pc = malloc(sizeof(PersonCell));

free(PersonCell);

  • Releasing is dangerous as you may free an in-use cell resulting in hard to detect errors → for this reason, not all programming languages allow you to do this.

Garbage collection

  • Java automatically finds unused objects i.e. nothing is pointing at them anymore.
  • If creation of object fails, heap will look for redundant cells and free them, finally it will attempt to make the object again.
  • The process may look like the following:
  1. Marking: Mark in-use objects, then free.
  2. Merging: If no free cell is large enough, merge adjacent free cells.
  3. Compaction: If still not enough, move all live cells to the bottom of the heap, leaving free space at the top.

Function reminder

  • malloc() → searches for free cell, if not found, more is requested from the OS.
  • free() → if adjacent cells to the free cell also free, if present, merges (coalesced) into a single but larger cell. If not found, the free list is just added to the free list for future.

Choosing a cell using malloc()

  • Best-fit: Smallest cell that is big enough to store required data.
  • First-fit: First cell that is big enough.
  • Next-fit: Same as best-fit but we remember where searching had finished and resume from that point on next use.

Priority Queues: Type II

  • Single queue of elements where each element contains information associated with object with its priority level.
  • Store elements in queue in decreasing priority order and remove highest (front).
  • Add: We scan to the correct position and then insert it there i.e. from beginning go past all elements with higher or same priority level.
  • Remove: No scanning required, simply remove first item then shuffle down.
  • Promotion: Increase each priority level by amount proportional to time item was waiting in queue → requires timestamping elements when added. Requires linear scan and then a sort to reorder queue using new priority values.

ADT: Graphs and Trees

Directed graph (digraph)

  • Good example is using networking, one person can be friends with someone, but they can be friends back e.g. Twitter.
  • If there is a link from one node to another, there may or may not be a link in th reverse direction.

Simple graph

  • Example includes the roads between towns and villages, you can go back and forth.
  • This would be a non-directed graph (simple graph) as there is no direction associated with a link i.e. simply connects node A to node B.
  • Distance is not essential to be a simple graph, it would just show whether connection exists.

Tree

  • The node at the top is called the root of the tree, trees are used to hold hierarchical information  - simply a graph with no loops i.e. no paths can rejoin.
  • Example, company A is divided into division C and M, M is further divided.

Graph representation

  • Represent graph as ordered pair specifying list of nodes (vertices, weight) and arcs (edges).
  • Node b is adjacent to another node if there is an arc (a, b) where a is the out-node and b is the in-node.
  • Arcs cannot be duplicated e.g. {(a, c), (a, c)} cannot exist, thus the maximum number of arcs is w2.
  • Graph that has maximum number of arcs is called a full graph, if none then empty. If only just a few then its density is sparse.
  • Density: Proportion of n2 possible which are present, thus

G = (N, A)

N = {a, b, c}

A = {(a, a), (a, c), (b, a), (c, a), (c, b)}

Complementary graph

  • Complement of graph G contains all the nodes of G and all the possible arcs that are not present in G → i.e. inverse.
  • Denoted as G*, thus density(G*) = 1 - density (G)
  • If one or more of the arcs of G* is deleted then it is a partial (reduced) graph.

Subgraph

  • Graph derived from G with one or more nodes deleted and all of the arcs deleted that were associated with the deleted node.

Paths and distances

  • Path of graph G is an ordered set of nodes such that each node is adjacent to its predecessor e.g. P = {(b, a), (a, c), (c, a), (a, a)}
  • Simple (elementary) path: Doesn’t visit any node more than once.
  • Loop (cycle): Path whose initial and final nodes are the same.
  • Simple loop: Loop that becomes a simple path if its final node is deleted.
  • Distance: Length of shortest path between two nodes.
  • Diameter: Largest distance that can be found in a graph.

Connectivity

  • In- and Out-Degree: Node has a degree (p, q) is p arcs point to and q arcs leave it where p is the in-degree and q is the out-degree.
  • Disconnected: Graph can be divided into two subgraphs (connected components).

e.g. All nodes of G are in either G1 or G2 and all the arcs of G are in either G1 or G2 with no duplicates in either n or a.

  • Connected: Strongly connected if atleast one path from every node connects to another. Otherwise, graph is weakly connected. If the graph is divided into subgraphs which are strongly connected then they are strongly connected components of graph.

Operations on graphs

  • Analysis: How many nodes or arcs, what is the diameter or density, what is the connection like, is the graph a tree?
  • Transformation: Form G*, then reduce, form a subgraph and delete nodes which cannot be reached from given start node (garbage collection).
  • Travelling: Carry out traversal of a graph, visit all nodes in some order.

ADT: Graph traversal

Traversal

  • Orderly visitation to every node in graph that can be reached from given start node, depending on its properties, it may not be possible to reach all other nodes.
  • Must record if nodes has been visited once already, otherwise get stuck in loop e.g. using flags or bit attached to node, or using array of flags.

Depth-first-traversal

  • Single path as far as possible, backtrack to previous junction and try alternative path when the way ahead is blocked.

Breadth-first-traversal

  • Visit nodes of increasing distance from start node, use queue to remember recently visited nodes e.g. visit nodes in order of hierarchy.

Spanning tree

  • We can highlight each movment to unvisited node to obtain reduced graph containing no loops.
  • Strongly connected digraph if we can travel from any node to any other.

Shortest path (Dijkstra algorithm)

  • Finds shortest paths from Node1 to Node2 in order of increasing distance from Node1.
  • Algorithm stops when reaches target Node or when it finds out that it can’t reach it.
  • Start from first node and branch to other nodes creating new arcs, algorithm will always choose the closest node in terms of arcs.

Minimal Spanning Tree

  • Each edge of the spanning tree has a weight or cost associated with it.
  • A minimum spanning tree refers to the spanning tree where the cost is no greater than any other spanning tree in the graph.

ALG: Tree traversals

Breadth-first-traversal

  • Going away from the root where we start at the top of the tree and make way down.
  • Going towards the root where we start from the bottom and make way up to root.

Depth-first-traversal (use example above)

  • Preorder: Visit node before visiting each of its subtrees in turn e.g. A, B, C, D, F, E.
  • Inorder: Visit the node’s first subtree, then the node, then the second subtree e.g. B, A, D, F, C, E.
  • Postorder: Each of node’s subtrees in turn and finally the node itself e.g. B, F, D, E, C, A.
  • Algorithms for traversals are all recursive, thus we need runtime stack to record where we have got to in pattern of methods calling methods.

ALG: Heap Sort

Overview

  • Control tree shape in order to as close as possible to balanced tree
  • Left-complete binary tree: Every level (row) except bottom is filled from left to right. Second node is always the left child of root, third node always right child.

Bubbling-up (sorting)

  • Have to bubble-up to the full height of the tree, so insertion is limited by tree height Log2(N+1) as heap-insertion is complexity class O(Log2N).
  • Thus to build min-heap of N nodes, primary efficiency is O(NLog2N).

Trickle-down (removal)

  • Swap value to be deleted with last node and then swap values until sorted.
  • Element must trickle down full height of tree so removal limited by tree height O(Log2N) as heap-retrieval is complexity class O(Log2N).
  • Thus to retrieve tree of N nodes, primary efficiency is O(NLog2N).

ALG: Sorting Techniques

Insertion Sort

  • Progress through array from left, increment until sorted.
  • Left segment is sorted (empty), right is not.
  • Take first element of right segment and insert it in the correct position of the left.
  • Has a complexity class of O(N2) based on worst case, average is N2.

Quick Sort

  • Pick an element X from the set of data
  • Put all elements < X on left, >= X on right and X at middle (partition).
  • Quicksort left then Quicksort right.
  • Has a complexity class of O(N2) based on worst case, average is NLog2N, thus faster than insertion.

Merge Sort

  • Split the list into equal halves.
  • Recursively sort each half.
  • Recombine by merging the halves.

Sorting algorithms comparison

Tree Sort

  • Builds a binary search tree (BST) from elements to be sorted and then traverses the tree in order so that elements come out in sorted order.
  • BST stores a key of each item stored in the tree which allows fast lookup, insertion and lookup.
  • Balancing the tree can help with searching as it improves the worst case.
  • Average case efficiency is NLog2N but worst case efficiency is N2.
  • Applications also include data compression and load balancing on parallel computers.
